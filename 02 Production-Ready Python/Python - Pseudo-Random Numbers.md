# Core: Pseudo-Random Numbers

## Summary

>Pseudo-random numbers are sequences of numbers generated by a deterministic mathematical formula that appear to be random. Because the process is deterministic, starting with the same initial value, called a 'seed', will always produce the exact same sequence of 'random' numbers, making experiments and simulations reproducible.

**Why This Matters:** Pseudo-random numbers are the bedrock of simulation, cryptography, and statistical sampling, enabling the creation of reproducible yet seemingly unpredictable outcomes from a deterministic machine.

_Analogy:_ _Think of a pseudo-random number generator as a complex recipe for a fruit salad. The 'seed' is a unique, secret starting ingredient, like a specific pinch of a rare spice. The 'mathematical formula' is the recipe itself—a fixed set of instructions (dice 3 apples, slice 2 bananas, add juice). Anyone who has the exact same secret ingredient (the seed) and follows the recipe (the algorithm) will produce the exact same fruit salad (the sequence of numbers) every single time. To an outsider who doesn't know the recipe or the secret ingredient, the resulting salad seems like a random assortment of fruit._

*   **Where it breaks down:** A recipe is designed to produce a consistent, predictable outcome. The goal of a pseudo-random number generator is the opposite: to produce a sequence that is as unpredictable and statistically random as possible, even though the underlying process is fixed. True randomness would be like grabbing handfuls of fruit from a giant bin with your eyes closed—there is no recipe and no way to replicate it.

## Details

How can a computer, a machine that strictly follows instructions, do something random? The short answer is, it can't. Instead, it fakes it using a clever process to generate pseudo-random numbers. This process uses a deterministic algorithm, meaning it's a predictable, step-by-step procedure. It starts with an initial value called a [[Python - Random Seed|seed]] and applies a mathematical formula to it to generate a number. That new number is then used as the input for the next calculation, creating a chain of numbers that pass statistical tests for randomness. This technique is fundamental to performing computational experiments, such as in [[Python - Hacker Statistics|Hacker Statistics]].

#### Primary Goal

To generate a sequence of numbers that appears statistically random but is fully deterministic and reproducible for the purposes of simulation, testing, and analysis.

#### Mechanism

- **How it Works:**
    1.  **Initialization:** The process begins with a number called the [[Python - Random Seed|seed]]. This is the starting point for the entire sequence.
    2.  **Generation:** A mathematical function, known as a pseudo-random number generator (PRNG), takes the current number (initially the seed) as input.
    3.  **Transformation:** The function performs a series of mathematical operations (like multiplication, addition, and modulus) to produce a new number.
    4.  **Iteration:** This new number becomes both the next 'random' number in the sequence and the input for the next round of the generation step. The process repeats, creating a long chain of numbers.
- **The Seed:**
    - This is the initial value that kicks off the generator. The entire sequence of numbers is completely determined by the seed. This direct link is explored in the [[Python - Random Seed & Reproducibility Relationship|relationship between the seed and reproducibility]].
    - *Example:* If you use `seed = 42`, you will always get the same sequence of numbers. If you use `seed = 123`, you will get a completely different, but also repeatable, sequence.
- **The Algorithm (Generator):**
    - This is the mathematical engine that drives the process. There are many different algorithms, each with different properties.
    - *Example:* A very simple (and not very good) example is a Linear Congruential Generator (LCG), which uses the formula: `X_n+1 = (a * X_n + c) mod m`, where `X_n` is the current number, and `a`, `c`, and `m` are pre-chosen constants.

##### Code Translation

```python
# A very simple demonstration of a Linear Congruential Generator (LCG)
# This is for educational purposes to show the deterministic mechanism.
# Modern libraries like NumPy use far more sophisticated algorithms.

# --- Parameters for the LCG algorithm ---
m = 2**32  # Modulus
a = 1664525  # Multiplier
c = 1013904223  # Increment

# --- Step 1: Initialization ---
# Set the starting seed value
seed = 42
current_number = seed

print(f"Starting with seed: {seed}")

# --- Steps 2-4: Generation, Transformation, Iteration ---
random_numbers = []
for _ in range(10):
    # Apply the formula to get the next number
    current_number = (a * current_number + c) % m
    # We can scale it to be between 0 and 1
    random_numbers.append(current_number / m)

print(random_numbers)

# If you run this code again with the same seed (42), you will get the exact same list of numbers.
```

#### Key Parameters

- **Seed Value:**
    - This is the primary lever for controlling the output. Setting a specific seed ensures [[Python - Reproducibility in Random Simulations|reproducibility]], which is vital for debugging code, validating scientific results, or ensuring fairness in a simulation like the [[Python - Empire State Building Walk Simulation Problem|Empire State Building walk]]. If no seed is set, systems often use the current time as a seed, resulting in a different sequence on each run.
- **Generator Algorithm:**
    - The choice of algorithm affects the quality of the random numbers. A good generator produces a sequence with a very long 'period' (the number of steps before the sequence repeats) and passes various statistical tests for randomness. Libraries like NumPy use robust and well-tested algorithms like the Mersenne Twister.

#### Core Trade-offs

- **Advantage: Reproducibility**
    - The greatest strength of pseudo-randomness is its deterministic nature. It allows developers and scientists to debug complex simulations and for others to verify their results by using the same seed. This is impossible with true randomness.
- **Disadvantage: Not Truly Random**
    - Because the sequence is generated by a formula, it is not truly unpredictable. Given the seed and the algorithm, the entire sequence can be recreated. This makes pseudo-random numbers unsuitable for applications where unpredictability is a security requirement, such as generating cryptographic keys.
- **Limitation: Periodicity**
    - Every pseudo-random sequence will eventually repeat itself. For good generators, this 'period' is astronomically large (e.g., 2^19937 - 1 for Mersenne Twister), making it a non-issue for most simulations. However, for poorly designed generators, the period can be short, leading to flawed and biased results.
