# Core: Pseudo-Random Numbers

## Summary

>A pseudo-random number is a number generated by a deterministic algorithm that appears to be statistically random. Unlike true random numbers, a pseudo-random sequence is entirely determined by an initial value called a [[Python - Random Seed|seed]], which means using the same seed will produce the exact same sequence of "random" numbers every time. This property is crucial for ensuring the [[Python - Random Seed & Reproducibility Relationship|reproducibility]] of experiments and analyses.

**Why This Matters:** Pseudo-random numbers are the cornerstone of reproducible scientific research and simulations, allowing anyone to verify and build upon analytical results by recreating the exact same conditions.

_Analogy:_ _Think of a pseudo-random number generator as a cookbook. The [[Python - Random Seed|seed]] is like choosing a specific recipe (e.g., "Recipe #42: Chocolate Chip Cookies"). The generator's algorithm is the set of instructions in that recipe. No matter who follows that exact recipe, they will always produce the same batch of cookies. The sequence of "random" numbers is the batch of cookies themselves—they seem unique, but their creation was perfectly predictable from the start._

**Where it breaks down:** A real recipe might have slight variations due to oven temperature or ingredient quality. A pseudo-random number generator is perfectly deterministic; given the same seed and algorithm, the output sequence is mathematically guaranteed to be identical, with zero variation.

## Details

The core idea is that while we often need randomness for tasks like simulations in [[Python - Hacker Statistics|Hacker Statistics]], true randomness is hard to come by and impossible to replicate. Pseudo-random numbers solve this by providing a sequence that *looks and feels* random for all practical purposes but is generated by a predictable, deterministic process. For any given starting point, or [[Python - Random Seed|seed]], the sequence of numbers that follows is fixed. This is incredibly useful because it ensures that your analysis or simulation is reproducible—anyone else can use the same seed and get the exact same results.

#### Primary Goal

To generate sequences of numbers that are statistically random enough for modeling and simulation, while being completely deterministic and reproducible for consistency and debugging.

#### Mechanism

- **How it Works:**
    1. **Initialization:** The process starts with an initial value called a [[Python - Random Seed|seed]]. This can be a user-specified integer or often derived from the system clock for a semblance of unpredictability if not specified.
    2. **Generation:** The seed is fed into a complex mathematical algorithm (the pseudo-random number generator or PRNG). This algorithm performs a series of calculations on the current number to produce the next number in the sequence.
    3. **Output:** The algorithm outputs a number that appears random. This new number then becomes the input for the next iteration, generating the subsequent number in the sequence.
- **The Seed:**
    - This is the starting point of the sequence. The entire sequence of numbers is uniquely determined by this initial value.
    - *Example:* If you set the seed to `42`, you will always get the same sequence of numbers from functions like `[[Python - numpy.random.rand()|numpy.random.rand()]]`.
- **The Algorithm (Generator):**
    - This is the deterministic engine that takes one number (the current state) and calculates the next. A common example is the Mersenne Twister algorithm, used by NumPy.
    - *Example:* The algorithm might be something conceptually similar to `next_number = (a * current_number + c) % m`, where `a`, `c`, and `m` are large, carefully chosen constants. The result `next_number` looks unrelated to `current_number`, but the process is perfectly defined.

##### Code Translation

```python
import numpy as np

# --- Step 1: Generate numbers without a fixed seed ---
# The seed is likely different each time you run this part,
# often based on the system clock.
print("Run 1 (no fixed seed):")
print(np.random.rand(3)) # Generates 3 random floats

print("\nRun 2 (no fixed seed):")
print(np.random.rand(3)) # Will produce DIFFERENT numbers than Run 1

# --- Step 2: Use a seed to ensure reproducibility ---
# By setting a seed, we fix the starting point of the algorithm.
print("\n--- Using a Fixed Seed ---")
np.random.seed(123)
print("Run 3 (seed=123):")
print(np.random.rand(3))

# --- Step 3: Reset the seed to get the same sequence again ---
# If we reset the same seed, the "random" sequence starts over.
np.random.seed(123)
print("\nRun 4 (seed=123 again):")
print(np.random.rand(3)) # Will produce the EXACT SAME numbers as Run 3
```

#### Key Parameters

- **Seed Value:**
    - This is the primary parameter controlling the entire sequence. Changing the seed value results in a completely different sequence of pseudo-random numbers.
    - A seed of `0` will produce one sequence, while a seed of `1` will produce another, entirely different one.

#### Core Trade-offs

- **Reproducibility vs. True Randomness:**
    - The main benefit is perfect reproducibility, which is essential for debugging code and validating scientific results. The cost is that the numbers are not truly random; they are predictable if you know the seed and algorithm.
- **Not Suitable for Cryptography:**
    - Because the sequence is predictable, standard PRNGs are insecure for cryptographic purposes like generating passwords or encryption keys. An attacker who figures out the seed can predict the entire sequence.
