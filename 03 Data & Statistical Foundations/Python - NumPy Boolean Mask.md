# Core: Boolean Masks

## Summary

>A Boolean mask is a NumPy array of `True` and `False` values that has the same shape as a target data array. It is generated by applying a logical condition to each element of the data array. This mask can then be used to perform [[Python - Filtering NumPy Arrays|element-wise filtering]], selecting only the elements from the original array where the corresponding mask value is `True`.

**Why This Matters:** Boolean masking is the cornerstone of efficient data filtering in NumPy, allowing you to select and modify subsets of data based on complex conditions without writing slow Python loops.

_Analogy:_ _Think of a Boolean mask as a stencil you lay over a piece of art. The art is your original NumPy array. The stencil has cutouts (representing `True`) and solid parts (representing `False`). When you spray paint over the stencil, the paint only goes through the cutouts, affecting only specific parts of the art underneath. Similarly, a Boolean mask "exposes" only the elements of the array you're interested in._

**Where it breaks down:** A stencil is a physical object, and you typically use it to apply something *new* (like paint). A Boolean mask in NumPy is a data structure used primarily for *selection* or *modification* of existing values, not just for applying a new, uniform value. The mask is also generated dynamically from the data itself, whereas a stencil is usually pre-made.

```
Original Array:  [ 1,  2,  3,  4,  5 ]
                     │   │   │   │   │
Condition:       (is element % 2 == 0 ?)
                     │   │   │   │   │
                     ▼   ▼   ▼   ▼   ▼
Boolean Mask:    [ F,  T,  F,  T,  F ]
(F=False, T=True)
```

## Details

The core idea behind Boolean masks is to create an intermediate array of `True`/`False` values by evaluating a condition across an entire NumPy array at once. This process, known as vectorization, is fundamental to NumPy's performance. For example, instead of looping through an array of numbers to find the even ones, you can apply the condition `array % 2 == 0` directly. This returns a new array—the mask—where each position is `True` if the number at that position in the original array is even, and `False` otherwise. This mask is the key to powerful and readable data selection techniques like [[Python - Filtering NumPy Arrays|filtering]] and is a common alternative to methods like [[Python - NumPy np.where() Function|np.where()]].

#### Primary Goal

To create a logical map (`True`/`False` values) that identifies which elements in a NumPy array satisfy a specific condition, enabling efficient, vectorized selection and modification.

#### Mechanism

- **Step 1: Define the Data Array**
    - Start with a NumPy array containing the data you want to evaluate.
- **Step 2: Formulate a Logical Condition**
    - Create a condition using comparison operators (e.g., `==`, `>`, `<`) and logical operators (e.g., `&` for AND, `|` for OR) that will be applied to each element.
- **Step 3: Generate the Boolean Mask**
    - Apply the condition directly to the NumPy array. NumPy automatically broadcasts the operation, evaluating the condition for every element and returning a new Boolean array of the same shape.

##### Code Translation

```python
import numpy as np

# --- Step 1: Define the Data Array ---
one_to_five = np.arange(1, 6)
print(f"Original Array: {one_to_five}")

# --- Step 2: Formulate a Logical Condition ---
# The condition is to check for even numbers (element % 2 == 0)

# --- Step 3: Generate the Boolean Mask ---
# Apply the condition to the entire array
is_even_mask = one_to_five % 2 == 0
print(f"Boolean Mask (is_even): {is_even_mask}")
```

#### Key Parameters

- **Comparison Operators**
    - The core of the condition is a comparison operator like `==` (equal), `!=` (not equal), `>` (greater than), `<` (less than), `>=` (greater than or equal), and `<=` (less than or equal).
- **Logical Operators**
    - For combining multiple conditions, you must use bitwise logical operators: `&` for element-wise AND, `|` for element-wise OR, and `~` for element-wise NOT. Standard Python `and`, `or`, `not` will not work as they can't operate on entire arrays.
    - Example: `(array > 5) & (array < 10)` creates a mask for numbers between 5 and 10.

#### Core Trade-offs

- **Pro: Performance and Readability**
    - Masking leverages NumPy's underlying C and Fortran code, making it significantly faster than explicit Python `for` loops (vectorization).
    - The syntax is often more concise and easier to read than a loop-based equivalent.
- **Con: Memory Consumption**
    - Creating a mask generates a new array in memory that is the same size as the original array. For very large arrays, this can be a significant memory overhead.

### Parent Concept

Boolean masking is a powerful method within the broader topic of [[Python - Indexing NumPy Arrays|indexing NumPy arrays]], providing a way to select elements based on their values rather than their position.

### Child Concepts

- The primary application of a Boolean mask is for [[Python - Filtering NumPy Arrays|filtering NumPy arrays]], where the mask is used inside square brackets to return a new array containing only the elements corresponding to `True` values.

### Related Concepts 

- This technique is often compared with [[Python - NumPy Fancy Indexing|fancy indexing]], which uses an array of integers to select elements at specific indices.
- The [[Python - NumPy np.where() Function|np.where() function]] provides an alternative, allowing you to return elements from one of two arrays based on a condition, which can sometimes replace the need for a two-step mask-then-filter process.
- The creation of the mask itself relies on [[Python - Boolean Operators on NumPy Arrays|element-wise Boolean operators]] like `&` and `|` to combine multiple conditions.
- When comparing the two main selection methods, [[Python - NumPy Fancy Indexing vs np.where|fancy indexing and np.where]] offer different approaches to conditional selection, each with its own performance characteristics.
