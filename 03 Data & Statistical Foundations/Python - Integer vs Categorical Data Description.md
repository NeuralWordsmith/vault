# Comparison: Describing Categorical Data

## Why This Comparison Matters

> The `.describe()` method in pandas intelligently adapts its output based on a column's data type. When a categorical column, such as 'marriage_status' encoded with integers, is incorrectly typed as numeric, `.describe()` produces misleading statistical summaries like mean and standard deviation. This is a classic example of [[DataEng - Dirty Data|dirty data]] leading to flawed analysis, embodying the [[DataEng - Garbage In, Garbage Out (GIGO)|'Garbage In, Garbage Out']] principle. After [[Python - Converting to Categorical Type|correctly converting the column]] to the 'category' data type, `.describe()` provides a much more appropriate summary, including the count of observations, the number of unique categories, the most frequent category (top), and its frequency (freq).

_Analogy:_ _Imagine asking a mechanic for a 'summary' of a car. If you tell them it's just a 'vehicle' (a generic, numeric-like type), they might give you its weight, length, and average speed—technically correct but not very useful for understanding what the car *is*. However, if you specify that it's a 'sports car' (a specific, categorical type), they'll instead tell you it has 2 doors, a V8 engine (the most frequent engine type for this model), and that only 500 were made (unique count). The summary's content changes to match the object's true category, providing far more relevant information._

**Where it breaks down:** A mechanic can often infer a car's type just by looking at it. A program like pandas cannot infer the *semantic meaning* of your integer codes (e.g., that the number `1` represents 'Single'). It relies entirely on the explicit data type you assign to the column to provide the correct type of summary.

## Side-by-Side Comparison

- **`.describe()` on Numeric Data**
    - Provides statistical summaries relevant to continuous variables.
    - Outputs include: `count`, `mean`, `std` (standard deviation), `min`, `25%` (1st quartile), `50%` (median), `75%` (3rd quartile), and `max`.
- **`.describe()` on Categorical Data**
    - Provides frequency and distribution summaries relevant to discrete variables.
    - Outputs include: `count` (number of non-null observations), `unique` (number of distinct categories), `top` (the most frequent category, or mode), and `freq` (frequency of the top category).

### Comparison Table

| Feature | `.describe()` on Numeric | `.describe()` on Categorical |
| :--- | :--- | :--- |
| **Purpose** | Summarizes central tendency and dispersion | Summarizes frequency and distribution of classes |
| **Key Outputs** | `mean`, `std`, `min`, `max`, quartiles | `unique`, `top` (mode), `freq` |
| **Use Case** | Age, salary, temperature | Gender, product category, country |
| **Misuse Risk**| Low, if data is truly numeric | High, if categorical data is left as an integer type |

## Key Similarities

Both variations are generated by the same pandas `.describe()` method and serve the common purpose of providing a high-level, condensed summary of a data column. Both outputs include the `count` of non-null observations as a fundamental metric.

## Verdict: When to Use Which

For columns representing discrete groups or labels (even if encoded as numbers), always convert to the 'category' type before using `.describe()` to get meaningful, non-mathematical summaries. For truly continuous numerical data, the default numeric summary is appropriate.

### Comparative Code Example
```python
import pandas as pd

# --- Step 1: Create a DataFrame with a mis-typed categorical column ---
# 'marriage_status' is categorical, but stored as integers (a form of dirty data).
data = {'age': [25, 30, 45, 35, 50],
        'marriage_status': [1, 2, 1, 3, 2]} # 1: Single, 2: Married, 3: Divorced
df = pd.DataFrame(data)

print("--- .describe() on Incorrect 'int' Type ---")
# This output is misleading because calculating the 'mean' of marriage statuses is nonsensical.
print(df['marriage_status'].describe())


# --- Step 2: Convert the column to the correct 'category' data type ---
# We use .astype() to perform the type casting, a key data cleaning step.
df['marriage_status'] = df['marriage_status'].astype('category')

print("\n--- .describe() on Correct 'category' Type ---")
# This output is aligned with categorical data, showing count, unique values, top, and frequency.
print(df['marriage_status'].describe())

# Expected Output:
# --- .describe() on Incorrect 'int' Type ---
# count    5.000000
# mean     1.800000
# std      0.836660
# min      1.000000
# 25%      1.000000
# 50%      2.000000
# 75%      2.000000
# max      3.000000
# Name: marriage_status, dtype: float64

# --- .describe() on Correct 'category' Type ---
# count     5
# unique    3
# top       1
# freq      2
# Name: marriage_status, dtype: int64
```
